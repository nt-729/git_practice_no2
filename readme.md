### 目的
git bash ではなく、主にgithubとVSCodeのみで運用してみる練習のリポジトリです

### やったこと
 - githubでリポジトリを作成
 - VSCodeからリポジトリを開く（URLをコピペして）
 - VSCodeからブランチの名前を「main」->「master」に変更
 - githubからもブランチの名前を変更
 - pull request を発行 -> 承認

### なぜそんなことしてる？
VSCodeだけで操作が完結しそうだったので、試してみた。  
ブランチの名前が「main」になった（デフォ値？）ので、「master」に変更しようとした。  
VSCodeでブランチ名を変更してreadmeも更新してプッシュしたところ成功したが、github上ではブランチ名が「main」のままだったので、githubでもブランチ名の修正をしてみた。  
するとVSCodeからファイルを修正してプッシュしたらできなくなっていて、githubを見たらpull requestの作成を促すような状態になってたので、プルリクエストしてマージした。  
その後、VSCodeでファイル修正→プッシュするができない状態になる。  
ローカルリポジトリを消して、クローンからやり直すことになった。

### 教訓
 - bashを使わないでVSCodeでリポジトリを作ってみたら、ブランチが「main」になった
 - リモートリポジトリのブランチ名の変更はVSCodeだけでは完結しない？？
